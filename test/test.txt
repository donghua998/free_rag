Spring Boot 三层架构详解

Spring Boot 是一个基于 Spring 框架的快速开发框架，它采用经典的三层架构设计模式。

第一层：Controller 层（控制层）
Controller 层负责接收前端发送的 HTTP 请求，并对请求参数进行校验。它不包含业务逻辑，只负责参数接收、校验和调用 Service 层。常用的注解有 @RestController、@RequestMapping、@GetMapping、@PostMapping 等。

第二层：Service 层（业务逻辑层）
Service 层是核心业务逻辑的所在地。它负责处理具体的业务规则、数据计算、事务管理等。Service 层通常定义接口和实现类，通过 @Service 注解标注，使用 @Transactional 管理事务。

第三层：DAO 层（数据访问层）
DAO 层负责与数据库交互，执行增删改查操作。在 Spring Boot 中，通常使用 MyBatis 或 JPA 来实现。MyBatis-Plus 提供了强大的 CRUD 功能，通过继承 BaseMapper 接口即可获得常用的数据库操作方法。

RBAC 权限模型介绍

RBAC（Role-Based Access Control）即基于角色的访问控制，是当前最广泛使用的权限管理模型。

核心概念：
- 用户（User）：系统的使用者
- 角色（Role）：权限的集合，如管理员、普通用户
- 权限（Permission）：具体的操作许可，如创建用户、删除文档

关系模型：
一个用户可以拥有多个角色，一个角色可以包含多个权限。这种设计使得权限管理变得灵活且易于维护。例如，当需要给某个岗位添加新权限时，只需修改该角色对应的权限即可，无需逐个修改用户。
