### 容器注意事项

#### Elasticsearch IK 分词器安装

ES 启动后需要安装 IK 中文分词器插件，否则中文全文检索无法工作。

```bash
# 1. 进入 ES 容器
docker exec -it kb-elasticsearch bash

# 2. 安装 IK 分词器（版本需匹配 ES 8.11.3）
elasticsearch-plugin install https://github.com/medcl/elasticsearch-analysis-ik/releases/download/v8.11.3/elasticsearch-analysis-ik-8.11.3.zip

# 3. 退出容器
exit

# 4. 重启 ES 容器使插件生效
docker restart kb-elasticsearch
```

**验证安装：**
```bash
curl -X GET "localhost:9200/_cat/plugins?v"
```

**注意事项：**
- IK 分词器版本必须与 ES 版本匹配
- 安装后必须重启 ES 容器
- 若网络下载失败，可手动下载 zip 后通过 `-v` 挂载到容器内安装

## 注意事项

1. **端口冲突**：MySQL使用3307，Milvus使用29530，ES使用9200，避免与本地其他服务冲突
2. **Docker服务启动顺序**：etcd → MinIO → Milvus → ES（ES 需额外安装 IK 分词器）
3. **权限注解**：使用 `@RequireRole` 或 `@RequirePermission` 标记需要权限的接口
4. **Token传递**：内部接口使用 `Authorization: Bearer <token>`（JWT）
5. **API Key传递**：外部接口使用 `X-API-Key` 请求头
6. **数据库初始化**：`src/main/resources/db/init.sql` 包含未建表的 DDL 脚本

## 架构设计说明

### 认证体系
- **内部（JWT）**：用户登录平台后，使用知识库 CRUD、文档管理、搜索等功能
- **外部（API Key）**：第三方系统通过 X-API-Key 调用搜索接口，返回知识片段
- API Key 只能访问创建者自己的知识库（通过 userId 校验）

### 检索体系
- **向量检索**（Milvus）：语义相似度搜索
- **全文检索**（Elasticsearch）：关键词匹配搜索
- **混合检索**（RRF融合）：两者结果加权排序，取互补优势

### QA 模块规划
- LLM 配置（provider / apiKey / model）存储在**用户账号**中
- 支持两种模式：
  1. 纯聊天：只调 LLM，不涉及知识库
  2. RAG：先搜索知识库，再结合 LLM 生成回答
- 对外 API Key 不包含 LLM 配置，仅提供搜索能力
